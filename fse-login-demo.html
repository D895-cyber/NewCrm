📱 What FSE Users See After Login

Step 1: FSE User Logs In

When an FSE user logs in, they are automatically redirected to the FSEMobileApp component.

FSE Login Flow
1. User enters FSE credentials 2. System detects user.role === 'fse' 3. Automatically shows FSEMobileApp 4. No need to navigate to #mobile-fse manually

Step 2: FSE Mobile App Interface

Step 3: Available Actions

FSE users can:

  • Start Workflow: Begin the step-by-step service process
  • Quick Report: Fill out ASCOMP service reports directly
  • View Schedule: See today's assigned visits
  • Check Reports: View recent service reports
  • Switch to Desktop: Access desktop dashboard if needed

🔧 Technical Details

Routing Logic

// In App.tsx if (user?.role === 'fse') { // Default: Show mobile FSE app for FSE users return <FSEMobileApp />; }

Component Structure

FSEMobileApp ├── Header (Welcome, Navigation) ├── Quick Actions (Start Workflow, Quick Report) ├── Today's Schedule ├── Recent Reports ├── Weekly Stats ├── Bottom Navigation └── Modals (ASCOMP Form, Download)

Data Flow

1. User logs in → App.tsx detects FSE role 2. FSEMobileApp loads → loadFSEReports() called 3. API calls made → Fallback to demo data if needed 4. UI renders with real or demo data 5. User can start workflow or fill reports
